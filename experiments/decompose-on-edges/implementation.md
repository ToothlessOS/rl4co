**Design choice (stage 3)**: We do not increase the per-node candidate
count. The pretrained `SparseGCNModel` is shape-locked to 20 candidates
per node (the per-node softmax in `NeuroLKH/net/sgcn_model.py:54` and
the embedding reshape at line 56 use `n_edges` as a hard dimension).
We instead **swap slot contents**: for each node `i`, the two edges
incident in the heuristic tour are forced into the candidate set,
displacing the two farthest 20-NN slots when the union exceeds 20.
The inverse edge index is recomputed in Python (see
`utils/neurolkh_runner._recompute_inverse_edge_index`, which matches
`NeuroLKH/SRC/FeatureGenerate.c:35-50` line-for-line). The augmented
graph builder is `utils/neurolkh_runner.build_tour_augmented_features`.

**Per-row augmentation** (matches the 2x3 layout):
- Row 0 (FI tour) → augmented graph forces in **FI** edges.
- Row 1 (LKH-2 tour) → augmented graph forces in **LKH** edges.

This keeps each tour "explained" by its own augmented model. Col-3 of
the figure (`NeuroLKH aug top-K`) follows the same top-K masking
convention as col-2 (`NeuroLKH top-K`); the only difference is the
*base* graph used for scoring. The headline metric is the **coverage
lift**: `(# tour edges in top-K on augmented) − (# tour edges in
top-K on 20-NN)`.
